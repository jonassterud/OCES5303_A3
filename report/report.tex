    \begin{abstract}
        Historically, oceanic eddies have been detected using ``Sea Level Anomolies'' (SLA) satellite images.
        However, the SLA resolution limits the accuracy - therefore, we'll deploy a ``Convolutional Neural Network'' (CNN)
        to detect eddies using other satellite formats without such limitations: ``Sea Surface Temperature'' (SST) and ``Chlorophyll-a'' (CHLA).
    \end{abstract}

    \section{The Data}

    \subsection{Overview}

    We have five different folders with data, retrieved from the ``Ocean Eddy Identification'' competition \cite{data:mercatorocean}:
    \begin{itemize} % bold? :(
        \item[] \texttt{\textbf{TRAIN\_PNG}} - Dated \verb|.png| satellite training images across various categories.
        \item[] \texttt{\textbf{TEST\_PNG}} - Dated \verb|.png| satellite testing images across various categories.
        \item[] \texttt{\textbf{LABELS\_TRAIN}} - Dated \verb|.csv| files with training labels.
        \item[] \texttt{\textbf{train\_images\_pkl}} - Dated folders, each containing various \verb|.pkl| files for the traning dataset.
        \item[] \texttt{\textbf{test\_images\_pkl}} - Dated folders, each containing various \verb|.pkl| files for the test dataset.
    \end{itemize}
    
    The \verb|.pkl| files contain various useful information, such as the contours and centres of the cyclonic and anticyclonic eddy's (in the training set).
    They also contain latitude and longitude positions of the images, which we'll use later for visualization.

    Additionally, we have labels for the test dataset as a \verb|.pkl| file.

    Our image labels are one of: \verb|0| (no eddies), \verb|1| (cyclonic eddies) or \verb|2| (anticyclonic eddies).

    \subsection{Parsing}

    We load, and parse, all the \verb|.pkl| files and put them into a dictionary.
    The \verb|.png| images are loaded in as grayscale images, and are then sorted under their respective dates, categories and image numbers - which are extracted from their filenames.
    
    Then, we combine all the images, and their corresponding metadata (from the \verb|.pkl| files) into a Pandas \verb|Dataframe|.

    \subsection{Splitting}

    We now have a \verb|DataFrame| with training data, which we use to train our model.

    Additionaly, we have a \verb|DataFrame| with test images.
    We split this up into two, randomly sampled, \verb|DataFrame|'s: a validation set and a seperate test set.
    
    The validation set is used to evaulate the performance of our models during model selection, and then, to get an unbiased metric on the
    generalization ability of our final model, we'll evaulate it on the test set.

    \subsection{Scaling and Resizing} \label{sec:scaling_and_resizing}

    There's different resolutions both inter- and intra-category between SST and CHLA.
    We need to resize these images in order to make the compatible with PyTorch's tensors.

    There's various methods to do this, most of which require some form of interpolation.
    But, given the low resolution, and the potential loss of quality, this might not be a good solution.

    Considering that we seem to be classifying images where the eddies are centered in the
    image (\ref{fig:cyclonic_eddies}) (\ref{fig:anticyclonic_eddies}), and that the images are \textit{almost} the same resolution,
    cropping the edges of the images seemed to be a better solution.

    Additionaly, the images are are in greyscale, with pixel values between 0 and 255.
    We'll scale these values before feeding them into our model, using a \verb|MinMaxScaler| to a range between 0 and 1.

    \subsection{Applying Filtering} \label{sec:applying_filtering}

    As showed previously (\ref{sec:filtering_images}), we have a way to filter out ``corrupt'' images.

    For SST, the reasoning for the corrupted images seems to be quite obvious, in that there's some interference from the land.
    Considering there's also so few samples, it seems quite safe to remove these from our training samples, but we'll test multipile implementations.

    However, for the CHLA images, we're filtering out approximately 92\% of the images, and removing these would most likely degrade the performance
    of our model quite severely. There's other options, such as masking out the corrupt images, e.g. by filling them with ones/zeros.
    We could also add a new channel, which specifies if an image is corrupt or not, e.g. a channel where all the pixels are ones/zeros marks for corrupt CHLA images
    and vice-versa. These approaches are also something we could do for the SST images.

    We'll try various techniques and see which ones perform well on our data

    \section{Convolution Neural Network}

    \subsection{Advantages}

    A ``Convolution Neural Network'' (CNN) is a feedforward neural network, with the purpose of learning features from an image.
    This involves a kernel, i.e. convolution matrix, being convoluted over the image matrix. We can specify a custom kernel size
    (i.e. amount of pixels being included in each convolution) and options such as stride and padding.

    \cite{enwiki:1349825931}
    \cite{dumoulin2016guide}

    \subsection{Configuration} \label{sec:cnn_configuration}

    For the CNN, we used three convolution layers. All of the convolution layers use a kernel size of three, and padding such that the input and output dimensions have
    the same dimensions. The first convolution layer transform the input into 16 channels, and each following convolution layer doubles the channel dimensions.
    Additionally, after convolution, we apply batch normalization, the ReLU activation function, and pooling.
    The two first layers use \verb|MaxPool2d| with a sliding $2 \times 2$ window - selecting the maximum value within its window.
    The final convolution layer, uses \verb|AdaptiveAvgPool2d|, which computes the mean within, unlike \verb|MaxPool2d|, dynamic regions.
    We use this to essentially flatten the last two feature dimensions.

    After the convolution layers, we have a fully connected layer, which halves the current amount of features.
    We then apply batch normalization, the ReLU activation function, and, to mitigate overfitting, a \verb|Dropout| layer with a probability set to 65\%.

    Finally, we have another fully connected layer, which scales down our features into three outputs (one for each label).

    \section{Model Selection}

    \subsection{Pipelines} \label{sec:pipelines}

    We test multipile pipelines, but all which have some default steps:
    \begin{itemize}
        \item Orientating the images (\ref{sec:orientation})
        \item Scaling and resizing the images (\ref{sec:scaling_and_resizing})
        \item Transforming into tensors.
    \end{itemize}

    The training dataset has some inbalance in the distribution between classes, we have approximately:
    \begin{itemize}
        \item No eddies: 38.15\%
        \item Cyclonic eddies: 32.76\%
        \item Anticyclonic eddies: 29.10\%
    \end{itemize}

    We therefore calculate balanced class weights, which are used in the \verb|CrossEntropyLoss|.
    The CNN filter weights are updated using \verb|AdamW|, and we adjust the learning rate using a cosine annealing schedule (with a minimum learning rate of $1\cdot 10^{-4}$).

    \section{Random Forest Classifier}

    \subsection{Advantages}

    Random forest's uses a collection of decision trees trained on random subsets of data,
    and chooses an output based on the majority vote (for classification) - they're known to being less likely
    to overfit, compared to standard decision trees.

    \cite{enwiki:1350010429}

    \subsection{Configuration}

    We used the trained convolution layers (\ref{sec:cnn_configuration}) in order to extract features from the images,
    and then trained a random forest classifier on the output from the convolution layers.

    Like when training the CNN pipeline (\ref{sec:pipelines}), we also specify that the random forest classifier should compute balanced class weights
    to use for its loss function.

    \subsection{Results}

    This approach achieved much better results on the validation data.
    It acheived an balanced accuracy of approximately 96\%.
